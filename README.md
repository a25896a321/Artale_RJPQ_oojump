## 🕹️ 操作流程與介面說明

### 1. 設定玩家

- 選擇**個人顏色**（點擊色塊）
- 輸入**暱稱**（1-10 位英數或中文字）
- 點擊「✅ 確定暱稱」

### 2. 建立房間（房主）

1. 點擊「🏠 建立房間」
2. 填寫房間名稱、密碼（選填）
3. 勾選選項：
   - **自動標記可能性**（預設開啟）：當隊友標記死路時，自動推算可能的玩家
   - **顯示成員名單**（預設開啟）：顯示當前連線成員，房主可踢出成員
4. 點擊「✨ 建立」
5. 點擊「🔗 邀請連結」將房間連結複製給隊友

### 3. 加入房間（成員）

- 方式 A：點擊房主分享的邀請連結，設定暱稱後點擊「加入」
- 方式 B：手動貼上房號至輸入框，點擊「加入」
- 若房間有密碼，將彈出密碼輸入框

### 4. 地圖互動

| 操作 | 電腦 | 手機 |
|------|------|------|
| 標記正確（每層限1格） | 左鍵點擊 | 單點 |
| 標記死路（每層限3格） | 右鍵點擊 | 雙擊 |
| 取消自己的標記 | 左鍵再點 | 再單點 |

### 5. 標記說明

- 🟢 **綠色（帶名字）**：某位玩家的正確格
- 🔴 **紅色（×N）**：已有 N 人標記為死路
- 🟡 **黃色閃爍（?）**：系統自動推算的可能格（顯示可能是哪些玩家）

### 6. 房間管理（房主專屬）

- 「🔑 編輯密碼」：修改或移除房間密碼
- 「🗑️ 清空所有標記」：清空整張地圖
- 「🔨 重建房間」：中斷所有連線並重新建立

### 7. 閒置保護

房間閒置超過 **1 小時**，將自動踢出所有連線並提示重建。

## 🌐 部署：GitHub + Cloudflare Pages

### GitHub 設定

```bash
git remote add origin git@github.com:a25896a321/Artale_RJPQ_oojump.git
git branch -M main
git push -u origin main
```

### Cloudflare Pages 串接步驟

1. 登入 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 進入 **Pages** → 點擊你的專案（已手動綁定 Git）
3. 進入 **Settings** → **Builds & deployments**
4. 設定如下：
   - **Framework preset**：`None`（純靜態 HTML）
   - **Build command**：（留空）
   - **Build output directory**：`/`（根目錄）
5. 點擊 **Save and Deploy**
6. 部署完成後，Cloudflare 會提供 `*.pages.dev` 網域

> ⚠️ 因為是純靜態 HTML，不需要任何建構指令。

## 📋 規則說明

- 每位玩家每層只能標記 **1 個正確格**
- 每格最多被標記 **3 個死路**（3人標誤 = 第4人必在此格）
- 房間上限 **4 人**
- 各玩家可自訂**暱稱**與**顏色**
