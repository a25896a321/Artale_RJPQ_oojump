## 🕹️ 操作流程與介面說明

### 1. 登入頁（首頁）
- 填寫**暱稱**（1-10 位英數或中文）
- 點擊色塊選擇**代表色**與**字體色**
- 選擇動作：

| 動作 | 操作 |
|------|------|
| **建立房間** | 填寫密碼（選填）、格子順序、功能選項 → 「✨ 建立房間」 |
| **加入房間** | 填寫房號（8位數）、密碼（如有）→ 「🔑 加入房間」 |
| **邀請連結** | 開啟含房號的連結 → 自動填入房號 → 填暱稱 → 加入 |

### 2. 房間介面
- **房間資訊**：顯示房號（可複製）、密碼（明碼顯示，可複製）
- **操作列**：🔑 編輯密碼 | 🔗 邀請連結 | 🚪 離開 | 🔨 重建 | ✏️ 暱稱
- **成員名單**：真實玩家 + 💀 替補機器人（灰色虛線標籤，代表缺席成員）

### 3. 地圖互動

| 操作 | 電腦 | 手機 |
|------|------|------|
| 標記正確（每層限 1 格） | 左鍵點擊 | 單點 |
| 標記死路（每層限 3 格） | 右鍵點擊 | 雙擊 |
| 取消自己的標記 | 再次點擊 | 再次點擊 |

### 4. 門格狀態說明

| 狀態 | 說明 |
|------|------|
| 正確標記 | 玩家個人色背景 + `O` + 玩家名 |
| ✗（紅色） | 自己標記為死路的格子（其他人不顯示 ✗） |
| ?(A/B) 紫框 | 系統推算該格可能的玩家 |
| ★僅剩此格 金框閃爍 | 唯一確定的玩家 |
| 💀A/💀B/💀C | 缺席替補，灰色虛線，納入可能性運算 |

### 5. Bot 替補機制
- 房間人數不足4人時，系統自動以 💀A、💀B、💀C 填補空位
- Bot 納入可能性推算，確保在人數不足情況下不會誤判「僅剩此格」
- 真實玩家加入時自動替換 Bot；玩家離開時 Bot 自動補位

### 6. 閒置保護
- 房間閒置超過 **1 小時**，Cloudflare Worker 自動清除房間並通知所有成員

## 🛠️ 技術架構

| 技術 | 版本 | 用途 |
|------|------|------|
| HTML5 / CSS3 / JavaScript | - | 純靜態前端（單頁應用） |
| Cloudflare Workers | - | WebSocket 伺服器入口、靜態資產服務 |
| Cloudflare Durable Objects | - | 房間狀態持久化、WebSocket 多連線管理 |
| Google Fonts (Noto Sans TC) | - | 繁體中文字體 |

### WebSocket 連線說明
- **建立連線**：每位玩家打開頁面時，瀏覽器與 Cloudflare 建立 WebSocket，消耗 **1 次請求額度**
- **互動過程**：所有訊息（標記、聊天、同步）**不計入額度**，走現有 WebSocket 管道傳輸
- **結束連線**：關閉頁面，WebSocket 中斷，Durable Object 進入休眠

## 🌐 部署方式

> ⚠️ v1.6.0 起改用 Cloudflare Workers + Durable Objects，需使用 **wrangler deploy** 部署，不支援純 Pages 自動部署。

### 首次部署
```bash
npm install wrangler --save-dev
npx wrangler login
npx wrangler deploy
```

### 後續更新
```bash
git add . && git commit -m "update"
npx wrangler deploy
```

### GitHub 設定（保存版本紀錄）
```bash
git remote add origin git@github.com:a25896a321/Artale_RJPQ_oojump.git
git branch -M main
git push -u origin main
```

## 📋 遊戲規則
- 每位玩家每層只能標記 **1 個正確格**（左鍵）
- 每位玩家每層最多標記 **3 個死路**（右鍵）
- 房間上限 **4 人**（不足時以 💀 Bot 補位，不影響推算）
- 各玩家可自訂**暱稱**與**顏色**
